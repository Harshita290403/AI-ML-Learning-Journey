1. Why Authentication is Important in APIs
   Authentication verifies the identity of a user or application before allowing access to an API.
   It is important because it:
    .Protects sensitive data from unauthorized users.
    .Ensures only valid users can access resources.
    .Prevents misuse of APIs.
    .Helps track user activity and maintain security.

2. What are Tokens?
   A token is a unique string generated by the server and given to an authenticated user.
   It acts like a temporary digital identity card.
    Example:
    Token 7f8d3a9b6c4e2f1g...
   Instead of sending a username and password with every request, the client sends the token.

3. How Token Authentication Works in DRF
   Step 1: User logs in
   The user provides a username and password.

   Step 2: Server validates credentials
   If the credentials are correct, the server generates or returns a token.

   Step 3: Client stores the token
   The token is stored by the client application (browser, mobile app, etc.).

   Step 4: Client sends the token
   For every protected API request:

     </>http:
       Authorization: Token 7f8d3a9b6c4e2f1g...

   Step 5: DRF verifies the token
    -Valid token → Access granted
    -Invalid token → Access denied

4. Real-World Examples of Token Authentication
   -GitHub API – Uses personal access tokens for accessing repositories.
   -Mobile Applications – Banking, shopping, and social media apps use tokens after login.
   -Food Delivery Apps – Users log in once and stay authenticated.
   -Streaming Platforms – Tokens maintain user sessions securely.

5. Difference Between Authentication and Authorization
     Authentication	                    Authorization
     Verifies who the user is	        Determines what the user can do
     Happens first                     	Happens after authentication
     Example: Logging in with           Example: Accessing an admin dashboard
             username and password	
     Confirms identity	                Grants permissions

     Example:
      Authentication: "Are you Harshita?"
      Authorization: "Can Harshita delete users?"

6. Why Shouldn't APIs Expose Sensitive Data Publicly?
   Public APIs without protection can lead to:
    .Data theft
    .Privacy violations
    .Unauthorized modifications
    .Account misuse
    .Security breaches
   Examples of sensitive data:
    -Passwords
    -Email addresses
    -Bank details
    -Personal information

7. What Happens if an Invalid Token is Sent?
   DRF rejects the request.
   Response:
     HTTP 401 Unauthorized
   Example:
   {
       "detail": "Invalid token."
   }
   The user must log in again or provide a valid token.

8. Steps to Configure Token Authentication in DRF
  Step 1: Install DRF
   pip install djangorestframework

  Step 2: Add required apps
  INSTALLED_APPS = [
     'rest_framework',
     'rest_framework.authtoken',
   ]

  Step 3: Run migrations
    python manage.py migrate

  Step 4: Configure authentication
   REST_FRAMEWORK = {
     'DEFAULT_AUTHENTICATION_CLASSES': [
         'rest_framework.authentication.TokenAuthentication',
     ]
 }

  Step 5: Generate tokens
   python manage.py drf_create_token username

  Step 6: Add token endpoint
   from rest_framework.authtoken.views import obtain_auth_token

  urlpatterns = [
      path('api-token-auth/', obtain_auth_token),
  ]

  Step 7: Use token in requests
   Authorization: Token your_token_here

12. Common HTTP Status Codes
    200 OK
    The request was successful.
    Example:
   {
      "message": "Student retrieved successfully"
   }

   401 Unauthorized
   Authentication is missing or invalid.
   Example:
   {
     "detail": "Authentication credentials were not provided."
   }

   403 Forbidden
   The user is authenticated but does not have permission to access the resource.
   Example:
   {
      "detail": "You do not have permission to perform this action."
   }
